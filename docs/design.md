# ボイス生成スタジオ デザイン定義書

## 1. デザイン方針

本ツールは、音声生成の作業画面である。装飾的なランディングページではなく、録音、参照音声登録、テキスト入力、音声生成、再生確認をすぐ行える実用画面を第一画面とする。

UIは暗色ベースで、長時間作業しても疲れにくく、操作対象が迷子にならない構成とする。

## 2. 画面構成

### 2.1 ヘッダー

- 左側にツール名「ボイス生成スタジオ」
- 補助テキストとしてローカル実行ツールであることを表示
- 右側にAPIキー設定、リセットボタンを配置
- APIキー設定では、保存方式と保存場所を選べるようにする

### 2.2 メイン領域

画面はデスクトップでは2カラムとする。

- 左カラム: 台本・テキスト編集
- 右カラム: ボイスクローン、音声生成、ブラウザ読み上げ
- 下部: 行ごとの生成結果確認

モバイルまたは狭い画面では1カラムに折り返す。

## 3. UIコンポーネント

### 3.1 台本パネル

- 見出し: `1. 台本`
- CSVフォーマット説明を小さく表示
- 作成形式として `1人用` / `複数人用` の選択を表示
- 大きなtextareaで編集
- 操作ボタン:
  - AI生成
  - サンプル
  - 台本を確認
  - CSV保存

### 3.2 ボイスクローンパネル

- 見出し: `2. ボイスクローン`
- OpenVoiceサーバー状態をステータス表示
- 同意チェックボックス
- 参照音声ファイル選択
- 録音開始・録音停止ボタン
- 参照音声プレビュー
- 声を登録ボタン

同意チェックは「本人の声、または利用許可を得た音声だけを使用します」と表示する。

### 3.3 音声生成パネル

- 見出し: `3. 音声生成`
- 生成テキスト入力欄
- 言語選択: 日本語、英語、中国語、韓国語
- 使用ボイス選択
  - 登録したクローン音声
  - 無料登録ボイス: 日本語
  - 無料登録ボイス: 英語
  - 無料登録ボイス: 中国語
  - 無料登録ボイス: 韓国語
- 音声を生成ボタン
- 生成音声プレイヤー
- 音声保存リンク

### 3.4 ブラウザ音声パネル

- 見出し: `4. ブラウザ音声`
- 読み上げ速度スライダー
- 声の高さスライダー
- ブラウザボイス選択
- 全て再生・停止ボタン
- OpenVoice未設定時の確認用途であることを補足表示

### 3.5 生成結果パネル

- 見出し: `5. 生成結果`
- 台本行数の要約
- 行ごとに以下を表示:
  - ボイスIDまたはポーズ
  - キャラクター名
  - セリフ本文
  - 単行再生ボタン

### 3.6 作成形式

- `1人用` は単独ナレーションの作業に最適化する
- `複数人用` は対談、会話、インタビュー形式に最適化する
- サンプルボタンは選択中の作成形式に応じたサンプルを読み込む

## 4. 色・トーン

### 4.1 カラーパレット

- 背景: `#111318`
- パネル: `#1b2029`
- サブパネル: `#141922`
- 境界線: `#303847`
- メインテキスト: `#f4f7fb`
- 補助テキスト: `#a8b1c0`
- アクセント: `#2f7fd8`
- 成功: `#2fba78`
- 警告・危険: `#c64848`

青系をアクセントに使うが、全体が単一色になりすぎないように、背景はニュートラルな黒灰色を中心にする。

### 4.2 角丸・余白

- パネル角丸: 10px
- 入力欄・ボタン角丸: 8px
- パネル内余白: 18px
- 主要グリッド間隔: 18px

## 5. タイポグラフィ

- ベースフォント: system-ui / Segoe UI
- 台本入力: Consolas または等幅フォント
- 見出しは過度に大きくせず、作業画面としての密度を保つ
- 文字間隔は標準とし、負のletter-spacingは使用しない

## 6. 状態表示

### 6.1 OpenVoice準備完了

- 緑系のステータス表示
- 文言例: `OpenVoice準備完了`

### 6.2 OpenVoice未起動

- 赤系のステータス表示
- 文言例: `OpenVoiceサーバー未起動。ブラウザ音声のみ利用できます`

### 6.3 OpenVoice未設定

- 赤系または警告系のステータス表示
- 文言例: `サーバー起動中。OpenVoice本体は未設定です`

### 6.4 処理中

- ボタンをdisabledにする
- 文言を `登録中...`、`生成中...` に変更する
- 完了またはエラーはtoastで通知する

### 6.5 APIキー保存設定

- 保存方式を選択できる
  - ブラウザに保存
  - ローカルファイルに保存
  - 保存しない
- ローカルファイル保存を選んだ場合は保存先パスを入力できる
- 保存先パスはユーザーが明示的に指定する
- 保存方式ごとに、再起動後の扱いを補足表示する

## 7. レスポンシブ

- 860px以下では1カラムに変更
- ボタンは折り返し可能
- 行カードは1カラム表示に変更
- 入力欄と音声プレイヤーは横幅100%

## 8. アクセシビリティ

- 主要操作はbutton要素を使う
- input、select、textareaにはlabelを関連づける
- 音声プレビューと生成音声にはブラウザ標準audio controlsを使う
- エラー通知は短文で表示する

## 9. 禁止・回避する表現

- 不要なランディングページ化
- 過度な装飾背景
- カードの入れ子構造
- 画面説明文の過剰配置
- 操作対象を曖昧にする抽象的なラベル
