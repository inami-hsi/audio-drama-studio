# ボイス生成スタジオ 要件定義書

## 1. 目的

本ツールは、ユーザー本人または利用許可を得た参照音声からボイスクローンを作成し、その声質を使って任意テキストの音声ファイルを生成するローカルファーストの音声生成ツールである。

初期段階では利用コストを抑えるため、OpenVoice V2、MeloTTS、ローカルPythonサーバー、ブラウザUIを中心に構築する。外部APIは、台本生成など必要な機能に限定して任意利用とする。

## 2. 対象ユーザー

- 自分の声でナレーション音声を作成したい個人ユーザー
- 許諾済み音声を使って教材、動画、SNS投稿用の音声を作りたい制作者
- 音声生成の検証を低コストで行いたい開発者

## 3. 利用前提

- 参照音声は本人の声、または明確に利用許可を得た音声のみ使用する
- 初期運用はローカルPC上で行う
- OpenVoice実行にはPython、PyTorch、MeloTTS、checkpointが必要
- CPU実行では生成速度が遅くなる可能性がある

## 4. 機能要件

### 4.1 音声入力

- ユーザーは参照音声ファイルをアップロードできる
- ユーザーはブラウザからマイク録音できる
- 参照音声は10秒以上を推奨する
- 参照音声を登録する前に、本人または許可済み音声であることへの同意チェックを必須とする
- 登録前に参照音声をプレビュー再生できる

### 4.2 ボイスクローン

- OpenVoice V2を使って参照音声から声特徴を抽出する
- 抽出した声特徴はローカルの `data/voice_profile` に保存する
- 既存の声特徴がある場合は、次回生成時に再利用できる
- OpenVoice未設定時は、UIに未準備状態を表示する

### 4.3 音声生成

- ユーザーは生成したいテキストを入力できる
- ボイスクローン作成後、音声生成では日本語、英語、中国語、韓国語を選択できる
- 作成形式として `1人用` と `複数人用` を選択できる
- `1人用` はナレーション、朗読、単独解説を想定する
- `複数人用` は対談、会話、インタビュー形式を想定する
- 複数人用ではキャラクターごとにボイスIDを割り当てられる
- クローン音声とは別に、サンプルまたは無料利用可能な登録ボイスを選択できる
- 無料登録ボイスは日本語、英語、中国語、韓国語のベース音声を用意する
- OpenVoiceサーバーが利用可能な場合、クローン音声でWAVを生成できる
- クローン音声未登録時でも、無料登録ボイスで音声生成できる
- 生成音声はブラウザ上で再生できる
- 生成音声はローカルに保存できる

### 4.4 台本支援

- CSV形式の台本を編集できる
- サンプル台本を読み込める
- 台本を行ごとに確認できる
- 台本CSVを保存できる
- Gemini APIキーを設定した場合のみ、AI台本生成を利用できる
- Gemini APIキーは設定画面で保存方式と保存場所を選択できる

### 4.5 フォールバック

- OpenVoice未設定または未起動時でも、ブラウザ標準音声でテキスト読み上げ確認ができる
- OpenVoiceサーバーの状態を画面上に表示する

## 5. 非機能要件

### 5.1 コスト

- 初期構成は無料で利用可能なOSSとローカル実行を優先する
- 外部APIは任意設定とし、ユーザー自身のAPIキー利用を基本とする
- サーバー常時稼働を前提にしない

### 5.2 セキュリティ・プライバシー

- 参照音声、声特徴、生成音声はローカル保存を基本とする
- 参照音声を外部サーバーへ送信しない
- APIキーはサーバーに保存しない
- 将来公開する場合は、認証、利用規約、アップロード制限、ログ管理を追加する

### 5.3 倫理・不正利用対策

- UI上で本人または許可済み音声のみ利用可能であることを明示する
- 同意チェックなしで声登録できないようにする
- 第三者のなりすまし、詐欺、誤認誘導を目的とした利用を禁止事項として明文化する

### 5.4 性能

- CPU環境では生成に時間がかかることを許容する
- 参照音声は短時間音声を想定する
- 長文生成は分割生成を将来対応とする

## 6. データ保存

- `data/references`: アップロードまたは録音した参照音声
- `data/processed`: OpenVoice前処理済み音声
- `data/voice_profile/target_se.pth`: 抽出した声特徴
- `outputs`: 生成されたWAV音声
- Gemini APIキー保存先:
  - ブラウザ保存: `localStorage.ads_gemini_api_key`
  - ローカルファイル保存: ユーザーが設定したローカルパス
  - 未保存: 起動中のみメモリ保持

## 7. 将来拡張

- 複数ボイスプロファイル管理
- 1人用/複数人用ごとのテンプレート管理
- 無料登録ボイスの追加、表示名、試聴
- 音声ごとの名前、用途、作成日時の保存
- 長文分割生成
- BGM、効果音、無音挿入
- Firebase Hosting公開
- Firebase Authによるログイン
- Cloud Functions経由のAPIキー保護
- 生成履歴一覧
- 商用利用時の同意ログ保存

## 8. 制約・リスク

- OpenVoice公式手順はLinux/Python 3.9を主対象としており、Windowsでは依存関係の調整が必要になる
- 初回セットアップでは大容量ダウンロードが発生する
- 音声品質は参照音声の録音品質に強く依存する
- CPUのみの環境では処理が遅い
- 悪用可能性があるため、公開運用時は利用者認証と同意ログが必要
